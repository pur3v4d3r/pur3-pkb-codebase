## âš ï¸ Important Notes

### Conflict Warning
Some modifiers override the same properties. If you enable multiple, the **last loaded** snippet takes precedence. Check snippet order in Settings â†’ Appearance â†’ CSS Snippets.

### Performance
- Most snippets are lightweight
- `callout-mod-03-neon-glow.css` (animations) may affect performance on slower devices
- Use `callout-mod-11-no-animations.css` if experiencing lag

### Theme Compatibility
- Designed for **dark themes** (optimal)
- Most work with light themes but may need opacity adjustments
- `callout-mod-09-retro-terminal.css` requires dark theme

### Adjust Intensity
Most modifiers use CSS variables. You can add your own snippet to override:

```css

  --callout-glass-opacity: 0.15 !important;
}
```

### Mix and Match
Create your own combination by:
1. Copying sections from different modifiers
2. Creating a new `callout-mod-13-custom.css`
3. Combining your favorite elements

### Color Tweaks
Your base `00-custom-callout.css` defines colors via `--callout-color` variable. Modifiers respect this, so changing colors in the base affects all modifiers.

## ğŸ†˜ Troubleshooting

### Modifier not working?
1. Ensure base `00-custom-callout.css` is enabled
2. Check snippet order (drag to reorder)
3. Click "Reload snippets" button
4. Restart Obsidian if needed

### Styles look weird?
- Check for conflicting snippets
- Try disabling all modifiers, then enable one at a time
- Some modifiers don't play well together

### Colors wrong?
- Modifiers inherit colors from base system
- Some modifiers (pastel, terminal) override colors intentionally
- Check your theme's color scheme compatibility
