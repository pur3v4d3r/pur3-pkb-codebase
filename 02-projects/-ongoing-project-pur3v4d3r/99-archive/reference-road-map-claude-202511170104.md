## âš ï¸ Common Pitfalls & Antidotes

> [!warning]
> **Critical Mistakes That Derail Progress**

### Pitfall 1: Perfection Paralysis
**Symptom**: Spending weeks designing perfect system before using it

**Reality**: Imperfect system in use beats perfect system in imagination

**Antidote**:
- Set a time limit: 2-3 hours for design, then build
- Accept 80% solutions
- Iterate based on actual use, not theoretical perfection
- Embrace "good enough for now"

### Pitfall 2: Feature Creep
**Symptom**: Adding new items to the roadmap instead of completing current ones

**Reality**: Completion is more valuable than variety

**Antidote**:
- Freeze scope for each phase
- Defer new ideas to next phase
- Finish before starting next
- Celebrate completion

### Pitfall 3: Skipping the Foundation
**Symptom**: Wanting to jump to Dataview queries or domain research before folder hierarchy is fixed

**Reality**: Automation of broken processes just makes brokenness faster

**Antidote**:
- Resist temptation to skip phases
- Remind yourself: every hour on foundation saves dozens later
- Use domain research as reward after phase completion

### Pitfall 4: Tool Obsession
**Symptom**: Constantly exploring new plugins, templates, methods

**Reality**: Master the fundamentals first

**Antidote**:
- Choose your stack: Dataview, Templater, QuickAdd (sufficient)
- Commit to using these tools for 3 months
- Only explore alternatives after mastery
- Document why you're considering changes

### Pitfall 5: Knowledge Hoarding
**Symptom**: Creating tons of notes but never synthesizing or using them

**Reality**: Unused notes are dead knowledge

**Antidote**:
- For every 5 new notes, create 1 synthesis
- Review permanent notes weekly
- Actively link new notes to existing ones
- Ask: "Why am I creating this note? How will I use it?"

### Pitfall 6: Cognitive Load Management Failure
**Symptom**: Trying to complete all phases simultaneously, burning out

**Reality**: You can't sprint a marathon

**Antidote**:
- Strict phase sequencing
- Realistic time estimates
- Regular breaks between phases
- Monitor energy levels
- Pause if you feel overwhelmed (re-read your cognitive load motivation)
