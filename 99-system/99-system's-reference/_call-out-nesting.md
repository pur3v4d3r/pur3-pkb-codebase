> 	1. Identity
> 	2. Current state
> 	3. Transversal Context
> 
> > > [!key] **Facet 1**: **The Type Axis** (#type/ or #type/reference/): *What is It?*
> >
> >
> > > [!important] **Implementation Examples:** 
> > > *Hierarchical tags define precise sources*:
> > > - `#resource/article`
> > > - `#resource/book`
> > > - `#resource/ebook`
> > > - `#resource/podcast`  
> > > 
> > > **For internally generated notes, tags may include**:
> > > - `#type/concept`
> > > - `#type/principle`
> > > - `#type/methodology`
> > 
> > 
> > > [!key] **Facet 2**: **The Status/Workflow Axis** (`#status/` or `#action/`) *Where does it stand in the process?*
> > 
> > 
> > >[!important] **Implementation Examples:**
> > > - `#status/draft`
> > > - `#status/triage`
> > > - `#status/review`
> > > - `#status/completed`
> > > 
> > >   
> > > **Action-oriented tags include**:
> > > - `#action/todo`
> > > - `#action/refactor`
> > > - `#action/priority/high`
> > > [!key] Facet 3: The Context/Attribute Axis (`#attribute/` or Lateral Modifiers) *How does it relate transversally?*
> > 
> > 
> > >[!important] **Implementation Examples:**
> > > - `#tool/dataview`
> > > - `#tool/python`
> > > - `#difficulty/high`
> > > - `#attribute/metaphor`
> > > - `#context/legal`.
