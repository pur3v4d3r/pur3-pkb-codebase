> [!definition]
> - **Key-Term**:: [[Tag Taxonomy]]
> - [**Definition**:: Tag taxonomy is a system for classifying and organizing information, often in digital environments, by using descriptive keywords or tags that are structured hierarchically or relationally.]

`Tags chosen for the system must be atomic, universal attributes (like #status/review), not broad, categories (belong to link/MOC structure)`

`(#a/b), it allows advanced Dataview to retrieve entire domains by selecting a top-level tag and automatically including all its subdivisions`

1. **Rule of Atomicity (Singularity):** 
	1. `Tags must represent the smallest semantic unit.` 
	2. ` (e.g., #books) must be replaced by (e.g., #book)`
2. **Case Consistency:** `All tags must use **all lowercase**`. 
3. **Format:** `Tags must contain **no spaces**.`

>[!tags-taxonomy]
> These describe a knowledge item in any PKB:
> 	1. Identity
> 	2. Current state
> 	3. Transversal Context
> 
> > > [!key] **Facet 1**: **The Type Axis** (#type/ or #type/reference/): *What is It?*
> >
> >
> > > [!important] **Implementation Examples:** 
> > > *Hierarchical tags define precise sources*:
> > > - `#resource/article`
> > > - `#resource/book`
> > > - `#resource/ebook`
> > > - `#resource/podcast`  
> > > 
> > > **For internally generated notes, tags may include**:
> > > - `#type/concept`
> > > - `#type/principle`
> > > - `#type/methodology`
> > 
> > 
> > > [!key] **Facet 2**: **The Status/Workflow Axis** (`#status/` or `#action/`) *Where does it stand in the process?*
> > 
> > 
> > >[!important] **Implementation Examples:**
> > > - `#status/draft`
> > > - `#status/triage`
> > > - `#status/review`
> > > - `#status/completed`
> > > 
> > >   
> > > **Action-oriented tags include**:
> > > - `#action/todo`
> > > - `#action/refactor`
> > > - `#action/priority/high`
> > > [!key] Facet 3: The Context/Attribute Axis (`#attribute/` or Lateral Modifiers) *How does it relate transversally?*
> > 
> > 
> > >[!important] **Implementation Examples:**
> > > - `#tool/dataview`
> > > - `#tool/python`
> > > - `#difficulty/high`
> > > - `#attribute/metaphor`
> > > - `#context/legal`.
