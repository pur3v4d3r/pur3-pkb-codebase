## ðŸŽ“ Tag Usage Guidelines

> [!core-principle]
> **Consistency Protocol:** Follow these rules to maintain taxonomy integrity:

### Rule 1: Minimum Tag Requirement
Every note in this domain MUST have:
- **One Type tag** (e.g., `#type/permanent`)
- **One Status tag** (e.g., `#status/evergreen`)
- **One or more Domain tags** (e.g., `#cognitive-science/metacognition`)

### Rule 2: Maximum Tag Density
- **Target:** 4-7 tags per note
- **Ceiling:** Never exceed 10 tags on a single note
- If you're hitting 10+, you're likely over-tagging or the note lacks focus

### Rule 3: Hierarchical Consistency
- Always use the **most specific** tag that applies
- If you tag `#cognitive-science/metacognition/monitoring`, you do NOT also need `#cognitive-science/metacognition` or `#cognitive-science` (Obsidian's search includes parent tags automatically)

### Rule 4: Cross-Cutting Concepts
- Use `#concept/` tags to capture specific cognitive phenomena that appear across domains
- A note can have multiple concept tags if it addresses multiple phenomena

### Rule 5: Context and Mode Tags
- `#context/` tags are OPTIONALâ€”use when the application context is significant
- `#mode/` tags are OPTIONALâ€”use for filtering by cognitive approach

### Rule 6: Source Attribution
- ALWAYS include a `#source/` tag for literature notes
- Consider adding author and date properties to frontmatter for literature notes

## ðŸ“– Semantic Definitions: Deep Dive

> [!definition]
> **Detailed Tag Meanings:** When in doubt, consult these definitions to ensure semantic consistency.

### cognitive-science/metacognition
**Definition:** The awareness and regulation of one's own cognitive processes. Includes monitoring, evaluating, and controlling one's thinking.

**Use for:**
- Notes about thinking about thinking
- Self-awareness of cognitive processes
- Monitoring comprehension or learning
- Reflective practices focused on cognition

**Do NOT use for:**
- Emotional self-awareness (use `#self-regulation/emotional`)
- Behavioral self-monitoring (use `#self-regulation/behavioral`)

---

### report/psychology/theory/self-determination
**Definition:** Edward Deci and Richard Ryan's theory of motivation emphasizing autonomy, competence, and relatedness as fundamental psychological needs.

**Use for:**
- Literature notes on SDT
- Analysis of intrinsic vs. extrinsic motivation
- Notes about autonomy-supportive environments
- Content addressing the three basic needs

**Do NOT use for:**
- General self-regulation content (use broader `#self-regulation` tags)
- Other motivation theories (use appropriate theory tag)

---

### topic/pshycology
**Definition:** The process of identifying, evaluating, and reconstructing arguments including their premises, conclusions, and logical structure.

**Use for:**
- Notes on identifying argument structure
- Evaluating argument strength
- Reconstructing implicit arguments
- Toulmin model, Aristotelian logic applied to arguments

**Do NOT use for:**
- General analytical thinking (use `#critical-thinking/analysis`)
- Mathematical or scientific analysis (use domain-specific tags)

---

### topic/psychology
**Definition:** Malcolm Knowles' theory of adult learning emphasizing self-direction, experience-based learning, and problem-centered (rather than content-centered) education.

**Use for:**
- Adult learning principles
- Self-directed learning strategies
- Experience-based education
- Distinguishing adult learning from child pedagogy

**Do NOT use for:**
- Child education (use `#learning-theory/pedagogy`)
- Self-regulated learning psychology (use `#self-regulation`)

---

### cognitive-science/cognitive-load
**Definition:** The total amount of mental effort being used in working memory, based on Sweller's Cognitive Load Theory.

**Use for:**
- CLT theoretical content
- Intrinsic, extraneous, and germane load
- Instructional design based on CLT
- Cognitive load management strategies

**Do NOT use for:**
- General mental fatigue or burnout (use appropriate wellness tags)
- Attention or focus (use `#cognitive-science/attention`)

```yaml
  - cognitive-science/metacognition
  - cognitive-science/metacognition/monitoring
```

**Good:**
```yaml
tags:
  - cognitive-science/metacognition/monitoring
```

**Why:** Obsidian's search automatically includes parent tags. Only tag the most specific level.

### âŒ Mistake 3: Confusing Tags with Links
**Bad:**
```yaml
tags:
  - self-determination-theory
  - cognitive-load-theory
  - growth-mindset-theory
```

**Good:**
```yaml
tags:
  - self-regulation/theory/self-determination
  
Related theories:
- [[Cognitive Load Theory]]
- [[Growth Mindset]]
```

**Why:** Specific theories should be separate notes with links, not tags. Tags describe categories, links describe relationships.

### âŒ Mistake 4: Context Overload
**Bad:**
```yaml
tags:
  - context/personal
  - context/professional
  - context/academic
  - context/teaching
```

**Good:**
```yaml
tags:
  - context/professional
```

**Why:** Choose the PRIMARY context, not all possible contexts.

### âŒ Mistake 5: Using Tags for Properties
**Bad:**
```yaml
tags:
  - author/daniel-kahneman
  - year/2011
  - pages/499
```

**Good:**
```yaml
tags:
  - type/literature
  - cognitive-science/reasoning
author: Daniel Kahneman
year: 2011
pages: 499
```

**Why:** Specific, unique metadata belongs in YAML properties, not tags.
